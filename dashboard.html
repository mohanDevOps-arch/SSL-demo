üîÑ Request Flow for /dashboard Route

Step 1: Browser sends HTTPS request to https://example.com/dashboard
Step 2: Reverse proxy receives encrypted request and terminates SSL
Step 3: Proxy matches /dashboard route to analytics-server
Step 4: Request forwarded to analytics-server via internal network
Step 5: Analytics server processes request and returns metrics data
Step 6: Proxy re-encrypts and sends response back to browser

üéØ Reverse Proxy Benefits

Single SSL Certificate: One certificate covers all routes (/admin, /dashboard, etc.)

Load Balancing: Distribute traffic across multiple backend servers

Security Layer: Hide internal server architecture from public

Centralized Logging: All requests logged at proxy level

Caching: Improve performance by caching responses at proxy
